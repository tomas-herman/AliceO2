8 
9 Local example workflow with local CCDB (running on port 8080) :
10 
11 This will read the list of DPs to be associated to TOF from CCDB (remove
12 `--use-ccdb-to-configure` if you don't want this, but use hardcoded

77 
78 By default the rate of the TF sampling will match to `mean-latence / lanes`, so that the sampling and generation are balanced and no TF is dropped out at the generation level.
79 One can provide the option `--pressure <float, D=1.>`, in which case the sampling rate will be increased by this factor, creating a back-pressure on the processing due to its latency.
80 
81 There is a possibility to generate data in multiple independent workflows, still avoiding TF-s with the same ID in different workflow. Options `--gen-norm <N, D=1> --gen-slot <I,D=0>` will
82 enforce skipping all TF-s except those with ``(TFID/lanes)%N == I``. I.e. for example, to emulate the processing with 3 different EPN's with 8 lanes each one can run in 3 terminals:
83 
84 ```shell
85 # Term.1: will produce TFs [0:7], [24:31], [48:55] ...
86 o2-calibration-data-generator-workflow --lanes 8 --max-timeframes 5000 --gen-norm 3 --gen-slot 0
87 
88 # Term.2: will produce TFs [8:15], [32,39], [56:63] ...
89 o2-calibration-data-generator-workflow --lanes 8 --max-timeframes 5000 --gen-norm 3 --gen-slot 1
90 
91 # Term.3: will produce TFs [16:23], [40:47], [64:71] ...
92 o2-calibration-data-generator-workflow --lanes 8 --max-timeframes 5000 --gen-norm 3 --gen-slot 2
93 ```
